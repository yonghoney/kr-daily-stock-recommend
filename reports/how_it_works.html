How it works · 초보자용 가이드

추천 결과는
마법이 아니라
순서 있는 계산입니다

VS Code에서 run_daily.py를 실행하면, 프로그램이 워치리스트를 자동 갱신하고 시세·수급·뉴스를 모아 점수표처럼 정리한 뒤 「매수관심 / 관망 / 주의」를 붙입니다. ChatGPT 같은 AI에게 물어보는 방식이 아닙니다.

API 키 불필요 자동매매 없음 최종 판단은 사람
아래로 스크롤하며 흐름을 따라가 보세요

한눈에 보는 전체 그림

식당에서 메뉴를 고를 때와 비슷합니다. 오늘 볼 메뉴판을 먼저 고르고, 재료(시세·수급)를 본 뒤 맛·상태(지표)를 체크하고, 점수표로 순위를 매깁니다.

입력
워치리스트 (자동)

config/kr_universe.yaml · 실행마다 갱신

출력
HTML 리포트

reports/daily/latest.html (+ txt·md·json)

단계별 흐름 — 결론이 나오기까지

프로그램은 아래 순서를 위에서 아래로 그대로 따릅니다. 중간에 AI가 “감으로” 끼어들지 않습니다.

01

시작 — run_daily.py 실행

VS Code에서 재생 버튼을 누르면 Python이 이 스크립트를 엽니다. Cursor나 채팅 AI와 연결되지 않아도 됩니다. 인터넷만 되면 시세·수급·뉴스를 받아올 수 있습니다. 뉴스를 빼고 빠르게 돌리려면 python run_daily.py --no-news 를 씁니다.

당신이 하는 일: 실행 버튼
02

워치리스트 자동 갱신

실행할 때마다 KRX 기준으로 관심 종목을 전체 교체해 config/kr_universe.yaml에 저장합니다. 코스피·코스닥 각각에 대해 (우선주 제외):

  • 시가총액 상위 50종목
  • 최근 7일 중 하루라도 거래대금 상위 20에 들어간 종목 중, 위 시총 50에 없는 종목

합집합(중복 제거)이 오늘의 분석 대상입니다. 대략 110~120종목 규모입니다.

비유: 오늘 볼 메뉴판을 새로 씀
03

시세·수급·펀더멘털 가져오기

각 종목에 대해 대략 다음을 모읍니다.

  • 시세 — Yahoo Finance OHLCV (최근 약 6개월). 장중이면 마지막 봉은 당일 미완성 시세일 수 있습니다.
  • 외국인·기관 순매수 — 네이버 추이 API (보통 전일·최근 수일 누적)
  • PER·PBR·시가총액·발행주식수 — 리포트 표시용 (점수에는 넣지 않음)

데이터가 너무 짧으면(25거래일 미만) 다른 시장 접미사(.KS ↔ .KQ)로 한 번 더 시도합니다. 그래도 부족하면 “데이터 부족”으로 표시합니다.

재료 준비: 가격·수급 기록장
04

기술 지표 계산

가격 숫자만으로 상태를 요약합니다.

  • 이동평균(20·60·120일) — 점수용 추세 판별 (종가가 이평 위/아래)
  • 수익률(1·5·10·20·60·120일) — 단기~장기 상승/하락 폭 (리포트에 표시)
  • RSI(14) — 과매수·과매도 느낌의 온도계(0~100)
  • 거래량 비율 — 평소보다 많이 거래됐는지
요리: 재료를 재고 간 보기
05

점수 매기기 + 이유 문장 만들기

미리 정해 둔 규칙표에 따라 점수를 더하거나 뺍니다. 기술지표가 본점수이고, 외국인·기관 수급은 그보다 작은 가중치(±3~8 정도)로 가산·감점합니다. 동시에 사람이 읽을 수 있는 이유 문장과 기여도가 큰 요인(드라이버)을 함께 남깁니다.

채점: 시험 답안지 채점과 비슷
06

뉴스 헤드라인 붙이기 (기본 ON)

최근 한국 뉴스 제목을 가져와 게시일과 함께 적고, 최신순으로 정렬합니다. 뉴스 내용으로 점수를 올리거나 내리지는 않습니다. “이런 기사가 있었다”는 참고용입니다. --no-news로 끌 수 있습니다.

곁들임: 양념은 참고만
07

등급 붙이기

총점으로 세 가지 중 하나를 고릅니다.

매수관심
점수 +25 이상 — 기술·수급 규칙상 상대적으로 우호
관망
중간 구간 — 뚜렷한 신호 없음
주의
점수 −20 이하 — 약세·과열 등 리스크 신호
결론 라벨 부착
08

리포트 저장

모든 종목을 점수 순으로 정렬해 저장합니다.

  • latest.html — 브라우저용 본문 (종합 요약 + 종목별)
  • latest.txt / .md / .json — 메모장·마크다운·기계 읽기용
  • YYYY-MM-DD/report.* — 같은 날 날짜별 보관본
출력: 오늘의 성적표

점수는 어떻게 올라가나요?

“좋다/나쁘다”를 AI가 느끼는 게 아니라, 아래 같은 가산·감점 규칙의 합입니다. 총점은 −100~+100 사이로 잘라 둡니다.

  • 종가가 20·60일 평균 위면 가점, 20일선 아래면 감점 / 20≥60이면 정배열 가점
  • 데이터가 충분하면 120일선 위·아래와 60↔120 배열도 가점·감점
  • 5일·20일 동안 많이 올랐으면 가점, 크게 빠졌으면 감점
  • RSI가 중간~건전 구간이면 가점, 너무 뜨거우면(과매수) 감점
  • 거래량이 평소보다 크게 늘며 상승하면 가점, 하락하면 감점
  • 외국인·기관 순매수/순매도(전일·최근 수일)는 기술보다 작은 가중치로 가점·감점

중요: 같은 규칙이라도 시장 전체가 약세면 워치리스트 대부분이 「관망」「주의」로 나올 수 있습니다. 그건 버그가 아니라, 오늘 데이터가 그렇게 채점된 결과입니다. 뉴스·PER·PBR·시가총액은 점수에 넣지 않습니다.

HTML 리포트에는 무엇이 있나요?

기본으로 볼 파일은 reports/daily/latest.html 입니다. 브라우저에서 세로로 스크롤하며 읽도록 만들어져 있습니다.

  • 종합 요약 — 점수 순 순위, 상위 드라이버, 종목 섹션으로 바로 가는 링크
  • 종목 헤더 — 이름·코드·시장 라벨(코스피/코스닥)·액션·점수
  • 핵심 수치 — 종가, 1일%, 5일%, RSI14
  • 기간 수익률·펀더 — 10/20/60/120일 상승률, PER, PBR, 시가총액, 발행주식수
  • 3개월 캔들 차트 — 국내식 색(양봉 빨강 / 음봉 파랑) + SMA 20·60·120
  • 수급 — 외국인·기관 1일·누적 순매수와 보유 비중(있을 때)
  • 뉴스 — 게시일 포함, 최신순 (점수 미반영)

JSON에는 차트 SVG를 넣지 않습니다(용량). 차트는 HTML에서만 보입니다. 날짜 폴더(reports/daily/YYYY-MM-DD/)에는 같은 내용의 보관본이 남습니다.

그리고… 사람이 하는 일

프로그램은 추천 메모장까지만 만듭니다. 돈은 움직이지 않습니다.

A

리포트 읽기

latest.html에서 순위표·차트·이유를 확인합니다. 메모장이면 latest.txt를 열면 됩니다.

B

의심하고 교차검증

공시, 실적, 본인 투자 원칙과 맞는지 직접 확인합니다. 점수는 참고일 뿐입니다.

C

직접 주문 (원할 때만)

증권사 앱·HTS에서 본인이 매수/매도합니다. 이 코드는 주문을 넣지 않습니다.

자주 생기는 오해

❌ “AI가 미래 주가를 안다”

아니요. 과거 가격·수급으로 만든 규칙 점수입니다. 내일 반드시 오른다는 뜻이 아닙니다.

❌ “자동으로 사고판다”

아니요. 리포트 파일만 만듭니다. 모의·실전 주문 연동은 켜져 있지 않습니다.

❌ “ChatGPT API가 필요하다”

기본 경로(run_daily.py)는 LLM API 키가 없어도 동작합니다.

✅ “매일 같은 절차로 갱신된다”

맞습니다. 실행할 때마다 워치리스트·시세·수급을 받아 같은 채점표로 다시 계산합니다.

❌ “뉴스가 점수를 바꾼다”

아니요. 뉴스는 참고용 헤드라인만 붙입니다. 점수에 반영되지 않습니다.

❌ “종가는 항상 어제의 확정가”

장중에는 마지막 봉이 당일 미완성 시세일 수 있습니다. 수급(외인·기관)은 보통 전일 기준입니다.

이 문서는 프로젝트의 일일 추천 파이프라인 (run_daily.py → tradingagents/recommend/)을 초보자용으로 설명한 보고서입니다. 투자 자문이 아닙니다.
