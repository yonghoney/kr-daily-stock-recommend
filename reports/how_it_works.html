VS Code에서 run_daily.py를 실행하면, 프로그램이 워치리스트를 자동 갱신하고 시세·수급·뉴스를 모아 점수표처럼 정리한 뒤 「매수관심 / 관망 / 주의」를 붙입니다. ChatGPT 같은 AI에게 물어보는 방식이 아닙니다.

단계별 흐름 — 결론이 나오기까지

프로그램은 아래 순서를 위에서 아래로 그대로 따릅니다. 중간에 AI가 “감으로” 끼어들지 않습니다.

01

시작 — run_daily.py 실행

VS Code에서 재생 버튼을 누르면 Python이 이 스크립트를 엽니다. Cursor나 채팅 AI와 연결되지 않아도 됩니다. 인터넷만 되면 시세·수급·뉴스를 받아올 수 있습니다. 기본 기준일은 마지막 확정 세션 종가입니다. 평일 15:30(KST) 이후면 당일, 장중·주말이면 직전 거래일입니다. 리포트 폴더명(YYYY/MM/DD)도 그 기준일과 같습니다. 뉴스를 빼고 빠르게 돌리려면 python run_daily.py --no-news 를 씁니다.

당신이 하는 일: 실행 버튼
02

누락 리포트 보완 (선택적 · 기본 켜짐)

매일 실행하지 못해도 괜찮습니다. reports/daily/.run_state.json에 마지막으로 채운 거래일을 저장해 두었다가, 다음 실행 때 그 이후~기준일 사이에 빠진 날짜만 찾습니다 (처음 한 번만 2025-01-01부터 검사). 빠진 날은 그날의 PIT 워치리스트로 리포트를 생성합니다. 건너뛰려면 --no-gap-fill.

자동 · 누락일만
03

워치리스트 자동 갱신

실행할 때마다 KRX 기준으로 관심 종목을 전체 교체해 config/kr_universe.yaml에 저장합니다. 코스피·코스닥 각각에 대해 (우선주 제외):

  • 시가총액 상위 50종목
  • 최근 7일 중 하루라도 거래대금 상위 20에 들어간 종목 중, 위 시총 50에 없는 종목

합집합(중복 제거)이 오늘의 분석 대상입니다. 대략 110~120종목 규모입니다.

비유: 오늘 볼 메뉴판을 새로 씀
04

시세·수급·펀더멘털 가져오기

각 종목에 대해 대략 다음을 모읍니다.

  • 시세 — Yahoo Finance OHLCV (최근 약 6개월). 장중이면 마지막 봉은 당일 미완성 시세일 수 있습니다.
  • 외국인·기관 순매수 — 네이버 추이 API (보통 전일·최근 수일 누적)
  • PER·PBR·시가총액·발행주식수 — 리포트 표시용 (점수에는 넣지 않음)

데이터가 너무 짧으면(25거래일 미만) 다른 시장 접미사(.KS ↔ .KQ)로 한 번 더 시도합니다. 그래도 부족하면 “데이터 부족”으로 표시합니다.

재료 준비: 가격·수급 기록장
05

기술 지표 계산

가격 숫자만으로 상태를 요약합니다.

  • 이동평균(20·60·120일) — 점수용 추세 판별 (종가가 이평 위/아래)
  • 수익률(1·5·10·20·60·120일) — 단기~장기 상승/하락 폭 (리포트에 표시)
  • RSI(14) — 과매수·과매도 느낌의 온도계(0~100)
  • 거래량 비율 — 평소보다 많이 거래됐는지
요리: 재료를 재고 간 보기
06

점수 매기기 + 이유 문장 만들기

미리 정해 둔 규칙표에 따라 점수를 더하거나 뺍니다. 기술지표가 본점수이고, 외국인·기관 수급은 그보다 작은 가중치(±3~8 정도)로 가산·감점합니다. 동시에 사람이 읽을 수 있는 이유 문장과 기여도가 큰 요인(드라이버)을 함께 남깁니다.

채점: 시험 답안지 채점과 비슷
07

뉴스 헤드라인 붙이기 (기본 ON)

최근 한국 뉴스 제목을 가져와 게시일과 함께 적고, 최신순으로 정렬합니다. 뉴스 내용으로 점수를 올리거나 내리지는 않습니다. “이런 기사가 있었다”는 참고용입니다. --no-news로 끌 수 있습니다.

곁들임: 양념은 참고만
08

등급 붙이기

총점으로 세 가지 중 하나를 고릅니다.

매수관심
점수 +25 이상 — 기술·수급 규칙상 상대적으로 우호
관망
중간 구간 — 뚜렷한 신호 없음
주의
점수 −20 이하 — 약세·과열 등 리스크 신호
결론 라벨 부착
09

리포트 저장

모든 종목을 점수 순으로 정렬해 저장합니다.

  • latest.html — 브라우저용 (종합 요약 중심 · 클릭 시 좌목록+우상세 · 시장/업종 필터)
  • latest.txt / .md / .json — 메모장·마크다운·기계 읽기용
  • YYYY/MM/DD/report.* — 같은 날 날짜별 보관본 (연 → 월 → 일)
  • signals.jsonl — 그날의 점수·액션을 백테스트 로그에 추가
출력: 오늘의 성적표

점수는 어떻게 올라가나요?

“좋다/나쁘다”를 AI가 느끼는 게 아니라, 아래 가산·감점 규칙의 합입니다. 기술지표가 본점수이고, 외국인·기관 은 그보다 작은 가중치로 더합니다. 최종 합은 −100~+100으로 자릅니다.

총점 ≥ +25
그 사이 (−19 ~ +24)
총점 ≤ −20

와 20·60·120일 이동평균의 위치·배열로 채점합니다. 은 그 평균값을 선으로 본 것입니다.

종가 < 20일선−15
20일선 ≥ 60일선 ( 성향)+10
20일선 < 60일선 ( 성향)−8
봉 ≥ 80개 · 종가 ≥ 120일선+8
봉 ≥ 80개 · 종가 < 120일선−8
봉 ≥ 80개 · 60일선 ≥ 120일선+4
봉 ≥ 80개 · 60일선 < 120일선−4

5일·20일 종가 대비 수익률입니다. (리포트에 보이는 10·60·120일 %는 표시용이며 점수에는 안 넣습니다.)

0~100 · 온도계입니다. 구간은 서로 겹치지 않게 하나만 적용됩니다.

HTML 리포트에는 무엇이 있나요?

기본으로 볼 파일은 reports/daily/latest.html 입니다. 처음에는 종합 요약이 중앙에 있고, 종목을 클릭하면 왼쪽 목록 + 오른쪽 상세로 나뉩니다.

  • 시장 점수 vs 지수 — 종합 요약 위에 코스피/코스닥별 종목 점수합·액션합(+1/0/−1)과 지수 비교 차트 (최근 3개월)
  • 종합 요약 — 점수 순 순위·상위 드라이버. 행을 클릭하면 상세 열림
  • 필터 — 시장(코스피/코스닥), 업종(네이버 업종명)으로 목록 좁히기
  • 종목 상세 — 오른쪽(모바일은 전체)에 메트릭·차트·수급·뉴스·점수 요인
  • 시그널 — 상단에 90일 매수관심/주의 횟수, 차트에 ▲매수·▼주의 표시
  • 점수 vs 종가 — 주가 차트 아래 최근 6개월 점수(좌)·종가(우) 이중축 차트
  • 점수대 통계 — reports/backtest/score_stats.html에서 1·3·6개월 선수익 참고
  • 핵심 수치 — 종가, 1일%, 5일%, RSI14
  • 기간 수익률·펀더 — 10/20/60/120일 상승률, PER, PBR, 시가총액, 발행주식수
  • 3개월 캔들 차트 — 국내식 색( 빨강 / 파랑) + SMA 20·60·120
  • 수급 — 외국인·기관 1일·누적 순매수와 보유 비중(있을 때)
  • 뉴스 — 게시일 포함, 최신순 (점수 미반영)

JSON에는 차트 SVG를 넣지 않습니다(용량). 차트는 HTML에서만 보입니다. 날짜 폴더(reports/daily/YYYY/MM/DD/)에는 같은 내용의 보관본이 남습니다.

백테스트는 어떻게 돌아가나요?

“그때 그 점수가 나중에 얼마나 맞았나”를 보기 위해, 그 날짜 이전 데이터만으로 점수를 다시 매기고(PIT), 이후 1·3·6개월 수익률을 점수대별로 모아 둡니다. 과거 성적표이지, 미래 수익 보장은 아닙니다.

1) PIT (Point-In-Time) — 미래 정보 차단

--as-of YYYY-MM-DD 또는 백필을 쓰면, 시세·수급을 그 날짜 당일 포함 이전만 사용합니다. 그다음 날 가격으로 점수를 만드는 “미래 엿보기”를 막습니다.

  • 단건: python run_daily.py --as-of 2026-07-10 --no-news
  • 기간: python run_backfill.py --from 2025-01-01 (거래일마다 그날 기준으로 워치리스트도 다시 만듦 · yaml은 덮어쓰지 않음)
  • 백필은 속도 때문에 뉴스·차트·펀더멘털 표시를 생략하고, 시세·기술·수급(가능 시)만으로 report.json을 쌓습니다. 워치리스트 일자 데이터는 FDR 일별 CSV가 없으면 FinanceData marcap 연간 파일을 내려받아 씁니다 (data/marcap/).

2) 시그널 로그 — signals.jsonl

매일(또는 백필 각 거래일) 종목별 날짜·코드·점수·액션·가격을 한 줄씩 reports/backtest/signals.jsonl에 남깁니다. 일일 실행 시에는 append, 긴 백필 뒤에는 날짜별 report.json에서 다시 모아 재구축할 수 있습니다.

3) 점수대 선수익 — score_stats

시그널 시점 종가 대비, 대략 거래일 기준 1개월(21일) · 3개월(63일) · 6개월(126일) 뒤 수익률을 계산하고, 액션 구간(주의 / 관망 / 매수관심)별 평균을 냅니다.

  • reports/backtest/score_stats.json — 기계 읽기용
  • reports/backtest/score_stats.html — 브라우저용 표 (리포트에서 링크)

표본이 적거나 보유 기간이 아직 안 지난 날은 평균이 비어 있을 수 있습니다. 네이버 수급 API는 최근 구간만 잘 나오므로, 먼 과거 날짜는 기술지표 위주로 채점된 날이 많습니다.

4) 리포트에 어떻게 쓰이나?

  • 차트 마커 — 과거 매수관심(▲)·주의(▼)가 찍힌 날
  • 90일 시그널 횟수 — 최근 매수관심/주의가 몇 번이었는지
  • 둘 다 signals.jsonl을 읽어 붙입니다. 백필을 안 돌렸으면 비어 있을 수 있습니다.
비유: 예전 시험 점수로 채점표가 잘 맞는지 점검

한계: 수수료·슬리피지·배당·상장폐지·워치리스트 변경을 전부 반영한 “실전 백테스트”는 아닙니다. 같은 규칙이 과거 점수대에서 어떤 선수익을 보였는지 참고 통계로 보세요.

리포트 읽기

B

의심하고 교차검증

공시, 실적, 본인 투자 원칙과 맞는지 직접 확인합니다. 점수는 참고일 뿐입니다.

C

직접 주문 (원할 때만)

증권사 앱·HTS에서 본인이 매수/매도합니다. 이 코드는 주문을 넣지 않습니다.

자주 생기는 오해

❌ “AI가 미래 주가를 안다”

아니요. 과거 가격·수급으로 만든 규칙 점수입니다. 내일 반드시 오른다는 뜻이 아닙니다.

❌ “자동으로 사고판다”

아니요. 리포트 파일만 만듭니다. 모의·실전 주문 연동은 켜져 있지 않습니다.

❌ “ChatGPT API가 필요하다”

기본 경로(run_daily.py)는 LLM API 키가 없어도 동작합니다.

✅ “매일 같은 절차로 갱신된다”

맞습니다. 실행할 때마다 워치리스트·시세·수급을 받아 같은 채점표로 다시 계산합니다.

❌ “뉴스가 점수를 바꾼다”

아니요. 뉴스는 참고용 헤드라인만 붙입니다. 점수에 반영되지 않습니다.

✅ “기본 실행은 확정 세션 종가까지”

맞습니다. 평일 15:30 이후면 당일 종가, 장중이면 전일 종가까지 씁니다. 폴더명도 그 기준일입니다.

❌ “백테스트 수익 = 앞으로 벌 돈”

아니요. 과거 점수대 평균 선수익일 뿐입니다. 수수료·체결·시장 국면 변화는 반영되지 않습니다.

✅ “과거 날짜는 그 시점 데이터만 쓴다”

맞습니다. --as-of·백필은 PIT로 잘린 시세·수급만 사용합니다.

이 문서는 프로젝트의 일일 추천 파이프라인 (run_daily.py → tradingagents/recommend/)을 초보자용으로 설명한 보고서입니다. 투자 자문이 아닙니다.
