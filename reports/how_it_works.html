How it works · 초보자용 가이드

추천 결과는
마법이 아니라
순서 있는 계산입니다

VS Code에서 run_daily.py를 실행하면, 프로그램이 관심 종목의 시세와 뉴스를 모아 점수표처럼 정리한 뒤 「매수관심 / 관망 / 주의」를 붙입니다. ChatGPT 같은 AI에게 물어보는 방식이 아닙니다.

API 키 불필요 자동매매 없음 최종 판단은 사람
아래로 스크롤하며 흐름을 따라가 보세요

한눈에 보는 전체 그림

식당에서 메뉴를 고를 때와 비슷합니다. 재료(데이터)를 보고, 맛·상태(지표)를 체크한 뒤, 점수표로 순위를 매깁니다.

입력
관심 종목 목록

config/kr_universe.yaml

출력
추천 리포트

reports/daily/latest.md

단계별 흐름 — 결론이 나오기까지

프로그램은 아래 순서를 위에서 아래로 그대로 따릅니다. 중간에 AI가 “감으로” 끼어들지 않습니다.

01

시작 — run_daily.py 실행

VS Code에서 재생 버튼을 누르면 Python이 이 스크립트를 엽니다. Cursor나 채팅 AI와 연결되지 않아도 됩니다. 인터넷만 되면 시세·뉴스를 받아올 수 있습니다.

당신이 하는 일: 실행 버튼
02

관심 종목 목록 읽기

config/kr_universe.yaml 파일에 적힌 종목 (예: 삼성전자, SK하이닉스…)을 하나씩 불러옵니다. 종목 코드를 Yahoo Finance 형식(005930.KS)으로 맞춥니다.

비유: 오늘 볼 메뉴판을 펼침
03

시세 데이터 가져오기

각 종목의 최근 약 6개월 주가·거래량을 받아옵니다. 데이터가 너무 짧으면(25거래일 미만) 다른 시장 접미사(.KS ↔ .KQ)로 한 번 더 시도합니다. 그래도 부족하면 “데이터 부족”으로 표시합니다.

재료 준비: 가격 기록장
04

기술 지표 계산

가격 숫자만으로 상태를 요약합니다.

  • 이동평균(20·60일) — 최근 평균 가격보다 위에 있는지
  • 수익률(1·5·20일) — 단기·중기 상승/하락 폭
  • RSI(14) — 과매수·과매도 느낌의 온도계(0~100)
  • 거래량 비율 — 평소보다 많이 거래됐는지
요리: 재료를 재고 간 보기
05

점수 매기기 + 이유 문장 만들기

미리 정해 둔 규칙표에 따라 점수를 더하거나 뺍니다. 예: “20일선 위면 +8”, “RSI가 과열이면 −15”. 동시에 사람이 읽을 수 있는 이유 문장을 함께 남깁니다.

채점: 시험 답안지 채점과 비슷
06

뉴스 헤드라인 붙이기 (선택)

최근 한국 뉴스 제목을 몇 개 가져옵니다. 뉴스 내용으로 점수를 올리거나 내리지는 않습니다. “이런 기사가 있었다”는 참고용으로만 적습니다.

곁들임: 양념은 참고만
07

등급 붙이기

총점으로 세 가지 중 하나를 고릅니다.

매수관심
점수 +25 이상 — 기술지표상 상대적으로 우호
관망
중간 구간 — 뚜렷한 신호 없음
주의
점수 −20 이하 — 약세·과열 등 리스크 신호
결론 라벨 부착
08

리포트 저장

모든 종목을 점수 순으로 정렬해 reports/daily/latest.md (및 날짜별 파일)에 저장합니다. JSON 파일도 같이 만들어 두어 나중에 표로 쓰기 쉽게 합니다.

출력: 오늘의 성적표

점수는 어떻게 올라가나요?

“좋다/나쁘다”를 AI가 느끼는 게 아니라, 아래 같은 가산·감점 규칙의 합입니다.

  • 주가가 20일·60일 평균보다 위면 가점 (추세가 우호적이라는 뜻)
  • 5일·20일 동안 많이 올랐으면 가점, 크게 빠졌으면 감점
  • RSI가 중간~건전 구간이면 가점, 너무 뜨거우면(과매수) 감점
  • 거래량이 평소보다 크게 늘며 상승하면 가점, 하락하면 감점

중요: 같은 규칙이라도 시장 전체가 약세면 워치리스트 대부분이 「관망」「주의」로 나올 수 있습니다. 그건 버그가 아니라, 오늘 데이터가 그렇게 채점된 결과입니다.

그리고… 사람이 하는 일

프로그램은 추천 메모장까지만 만듭니다. 돈은 움직이지 않습니다.

A

리포트 읽기

latest.md에서 순위표와 종목별 이유를 확인합니다.

B

의심하고 교차검증

공시, 실적, 본인 투자 원칙과 맞는지 직접 확인합니다. 점수는 참고일 뿐입니다.

C

직접 주문 (원할 때만)

증권사 앱·HTS에서 본인이 매수/매도합니다. 이 코드는 주문을 넣지 않습니다.

자주 생기는 오해

❌ “AI가 미래 주가를 안다”

아니요. 과거 가격으로 만든 규칙 점수입니다. 내일 반드시 오른다는 뜻이 아닙니다.

❌ “자동으로 사고판다”

아니요. 리포트 파일만 만듭니다. 모의·실전 주문 연동은 켜져 있지 않습니다.

❌ “ChatGPT API가 필요하다”

기본 경로(run_daily.py)는 LLM API 키가 없어도 동작합니다.

✅ “매일 같은 절차로 갱신된다”

맞습니다. 실행할 때마다 최신 시세를 받아 같은 채점표로 다시 계산합니다.

이 문서는 프로젝트의 일일 추천 파이프라인 (run_daily.py → tradingagents/recommend/)을 초보자용으로 설명한 보고서입니다. 투자 자문이 아닙니다.
